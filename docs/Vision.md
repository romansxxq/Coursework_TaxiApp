# Vision Document — TaxiApp (DB + Demo Web)

## 1. Короткий опис продукту
**TaxiApp** — інформаційна система для сервісу таксі, що забезпечує зберігання та обробку даних про пасажирів, водіїв, автомобілі, тарифи, поїздки та відгуки. Проєкт включає реляційну БД та демо вебінтерфейс для демонстрації роботи системи.

## 2. Проблема
Сервіс таксі потребує централізованого та надійного зберігання даних, швидкого доступу до інформації для інтерфейсів і формування звітів/аналітики.

## 3. Мета та цілі
### Мета
Спроєктувати та реалізувати реляційну базу даних для мобільного застосунку таксі та створити вебінтерфейс для демонстрації її роботи.

### Цілі
- Зберігати дані про користувачів, водіїв, авто, тарифи, замовлення та відгуки.
- Забезпечити реєстрацію/авторизацію користувачів і водіїв.
- Дати можливість створювати поїздки, переглядати історію, залишати відгуки.
- Підготувати базові аналітичні запити/звіти.

## 4. Зацікавлені сторони (Stakeholders).
- **Пасажир**: створює замовлення, переглядає історію, залишає відгук.
- **Водій**: приймає поїздки, оновлює статуси, переглядає оцінки/рейтинг.
- **Адміністратор** (опційно): переглядає аналітику та загальні списки.

## 5. Обсяг проєкту (Scope)
### In scope
- ER-модель і реляційна схема БД.
- DDL-скрипт створення таблиць і зв’язків.
- Демо вебінтерфейс: авторизація/реєстрація, робота з поїздками, тарифи, історія, відгуки, базові звіти.

### Out of scope (не робиться в межах курсової)
- Інтеграції з картами/геолокацією в реальному часі.
- Платежі та платіжні системи.
- Push/SMS сповіщення.

## 6. Основні можливості (High-level features)
- Реєстрація та авторизація пасажира/водія.
- Створення замовлення: адреса подачі, адреса прибуття, тариф.
- Керування статусом поїздки (new/accepted/in_progress/completed/canceled).
- Історія поїздок.
- Відгуки та рейтинг водія.
- Аналітичні запити (популярність тарифів, середні оцінки тощо).

## 7. Обмеження та припущення
- База даних: PostgreSQL.
- Сервер: Node.js + Express.
- Представлення: EJS templates.
- Паролі зберігаються у вигляді хешу.
- Проєкт демонстраційний.

## 8. Критерії успіху
- БД розгортається з DDL-скрипта без помилок.
- Працює сценарій: реєстрація → створення поїздки → перегляд історії → відгук.
- У репозиторії є документація: Vision, BRD/NFR, схема БД, деплой, README.

## 9. Ризики
- Витік секретів, якщо комітити `.env`.
- Неконсистентність даних без FK/constraints.
- Брак тестових даних для демонстрації.
