# Вимоги до системи (FR + NFR) — TaxiApp

## 1. Функціональні вимоги (FR)

### FR-1 Реєстрація та авторизація
Система повинна підтримувати реєстрацію та авторизацію користувачів (пасажирів) і водіїв.

### FR-2 Управління поїздками (замовленнями)
- Пасажир повинен мати можливість переглядати активні та минулі замовлення.
- Система повинна зберігати детальну інформацію про кожне замовлення.

### FR-3 Довідники: Водії / Автомобілі / Тарифи
- Система повинна зберігати та надавати інформацію про автомобілі, прив’язані до водіїв.
- Система повинна зберігати інформацію про тарифи для розрахунку вартості поїздки.

### FR-4 Відгуки
Система має підтримувати можливість користувачам залишати відгуки на водія після поїздки.

## 2. Нефункціональні вимоги (NFR)

### NFR-1 Продуктивність
Система має забезпечувати швидкий доступ до інформації.

### NFR-2 Надійність та коректність
Система повинна коректно обробляти та зберігати дані; робота БД і вебінтерфейсу має бути протестована.

### NFR-3 Безпека
Потрібно забезпечити захист даних користувачів через авторизацію з використанням хешування.

### NFR-4 Масштабованість
Архітектура БД має бути спроєктована з розрахунком на майбутнє розширення (наприклад, додавання налаштувань/уподобань/рекомендацій).
